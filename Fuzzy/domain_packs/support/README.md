# Fuzzy Support Pack

This sample optional pack shows the expected structure for a focused domain package.

It intentionally reuses the existing support-triage theme from the core repository as a contribution-standard example.

Package layout:
- `pyproject.toml` with standalone package metadata
- `src/fuzzy_support/` for exportable recipe code
- `compatibility.json` for Fuzzy compatibility and exported modules
- `evals/` and `tests/` for quality gates
